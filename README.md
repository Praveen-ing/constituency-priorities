# Constituency Priorities — Data Intelligence & Acceleration

> **Hackathon**: Google Cloud — Build with AI: Code for Communities  
> **Track**: Problem Set 1 (Data Intelligence & Acceleration)

A data-intelligence platform designed for Members of Parliament (MPs) to make faster, data-driven decisions on constituency development. It ingests civic complaints, cross-references them with public datasets, and uses an AI-powered **Gap Score** to rank the most critical areas needing investment.

This project was specifically pivoted to tackle **Track 1** by integrating **NVIDIA RAPIDS (cuDF)** and massive dataset simulation, demonstrating how GPU acceleration solves the computational bottlenecks of large-scale data intelligence.

---

## 🚀 Key Features Implemented (Why they exist & How to use them)

### 1. The Citizen PWA (Progressive Web App)
- **Why it's there:** To simulate the real-world ingestion of data. Citizens need a low-friction way to report issues.
- **What it does:** A mobile-first, installable web app that allows citizens to submit complaints via text. Includes a beautiful animated pipeline showing the journey of their data into the MP's dashboard.
- **How to use:** Navigate to `http://localhost:3000/citizen` in your browser. Type a mock complaint (e.g., "The school in Rajapuram is overcrowded") and click Submit. Watch the data processing animation.

### 2. The MP Dashboard & Dynamic Gap Score
- **Why it's there:** MPs receive thousands of requests and have limited budgets. They need a decision-support tool that ranks priorities based on hard data (Volume, Urgency, Data Deficit/Supply Gap, and Population affected).
- **What it does:** Displays a ranked list of constituency priorities. Each priority has a Gap Score and a plain-English justification generated by Gemini 2.5 Pro. It includes interactive "Weight Sliders" allowing the MP to adjust the importance of Urgency vs Volume on the fly.
- **How to use:** Navigate to `http://localhost:3000/dashboard`. View the ranked list. Open the "Adjust Weights" panel and move the sliders.

### 3. The NVIDIA GPU Acceleration "Demo Moment" (Track 1 Winner)
- **Why it's there:** To explicitly fulfill the Track 1 requirement: *"show how acceleration helps them make a faster or better decision."* When an MP adjusts the weight sliders, the Gap Score must be recomputed dynamically across the entire historical dataset. On a 5 million row dataset, standard CPUs choke, freezing the UI.
- **What it does:** We built a custom `generate_massive_dataset.py` script to simulate 5 Million rows of historical civic data. We also implemented `cudf.pandas` (NVIDIA RAPIDS). The dashboard features a toggle switch between CPU and GPU.
- **How to use:** On the dashboard, open the Weights panel. Leave the toggle on **CPU** and click Recompute (it simulates/runs the slow pandas bottleneck). Flip the toggle to **NVIDIA GPU** and click Recompute again for instantaneous results.

### 4. "Ask the Data" Chatbot (Gemini 1.5 Pro Integration)
- **Why it's there:** Dashboards are static; MPs often have ad-hoc questions about their data that a UI can't answer.
- **What it does:** A chat interface embedded in the dashboard that allows the MP to ask natural language questions (e.g., "Which ward has the most water complaints?"). The backend injects the current data context into a Gemini 1.5 Pro prompt to generate intelligent, data-backed answers.
- **How to use:** On the dashboard, click the Chat icon in the top right. Ask a question about the current priorities.

### 5. PDF Briefing Export
- **Why it's there:** Bureaucracy still runs on paper. MPs need to take these digital insights into physical meetings.
- **What it does:** Generates a professional, formatted PDF report of the current top priorities.
- **How to use:** Click the "Export Report" button on the dashboard.

## 🛠️ Architecture

```
Citizens (PWA)
        ↓
Ingestion Service (FastAPI)
        ↓
Data Fusion & Scoring (NVIDIA RAPIDS / cuDF)
  · Joins citizen submissions with public data
  · Computes deterministic Gap Score per theme × ward over 5M rows
        ↓
NLP / Reasoning (Gemini 1.5 Pro)
  · Gap Score justification string generation
  · "Ask the Data" natural language Q&A
        ↓
Frontend (Next.js 15)
  · Citizen PWA: submission pipeline
  · MP Dashboard: ranked priorities, weight sliders, GPU toggle
```

## 🏃‍♂️ Quick Start (Running Locally with No Blockers)

We have built a robust mock layer so the entire project runs locally **without needing any Google Cloud, Firebase, or Gemini API keys configured**.

### Prerequisites
- Python 3.12+
- Node.js 20+

### Step 1: Start the Backend & Seed Local Data
The backend uses a local SQLite database when GCP credentials are not present.

```bash
cd backend
python -m pip install -r requirements.txt

# Run this ONCE to seed the local DB with demo data and pre-computed Gap Scores
python -m app.db.seed_local_db

# (Optional) Run this ONCE if you want to generate the 5M row Parquet file for RAPIDS
python -m app.data.generate_massive_dataset

# Start the server
python -m uvicorn main:app --reload --port 8000
```
*Verify: `http://localhost:8000/priorities` should return JSON data.*

### Step 2: Start the Frontend
The frontend is pre-configured to bypass Firebase Auth locally and auto-login as a "Dev MP User".

```bash
cd frontend
npm install

# The .env.local is already created pointing to localhost:8000
npm run dev
```

### Step 3: View the Apps
- **MP Dashboard:** `http://localhost:3000/dashboard`
- **Citizen App:** `http://localhost:3000/citizen`
